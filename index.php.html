3. 깃허브 업로드

이제 제작한 코드를 업로드할 차례입니다. 깃허브 사용은 실제 컴퓨터공학 전공 학생들도 처음 접할 때는 무척이나 헷갈려하니 (저도 처음에는 헷갈렸어요!) 차근차근 하나하나 해봅시다. 



3-1. GitHub (https://github.com/)에 회원가입을 합니다. 



3-2. 이후 가입이 완료되었으면, 새로운 저장소 (repository)를 만듭니다. 버튼은 가입한 페이지의 우측 상단에 있습니다. 






3-3.  저장소를 만들었다면 깃허브 데스크탑 (https://desktop.github.com/) 을 설치합니다.

왜냐면... CLI는... 갑자기 허들이 너무 높아지니깐...ㅎ... 







3-4. 설치가 끝났다면 GitHub 아이디로 로그인합니다. 









3-5. 이후 프로그램을 실행하면 다음과 같은 화면이 뜹니다. your repositories 목록에서 아까 생성한 저장소를 찾아 클릭한 후 clone 버튼을 눌러주세요. 







3-6. 작업을 진행할 폴더 (어디든 상관 없습니다.) 를 고른 후에 Clone 버튼을 눌러주세요. 







3-7. 아까 선택한 경로에 저장소 이름과 똑같은 폴더가 새로 생겼다면 성공입니다. 



Q. 생성이 안 됐는데 어쩌죠? 

A1. 경로(저장소 폴더를 만들 폴더)를 제대로 지정하셨나요? 

A2. 제대로 로그인 하셨나요? (저장소를 만들어둔 계정으로)

A3. 제대로 저장소를 고르셨나요? (다른 이름의 저장소는 아닐까요?) 


이래도 안 된다면... 다시 3-2번(repository 생성)을 진행해봅시다. 







3-8. 이후 생성된 저장소 폴더 안에 아까 daxigua-master 폴더 안의 내용들을 넣어줍니다. 

폴더채로 넣으면 안 됩니다! 폴더 안의 내용을 전부 넣어주셔야해요! 








3-9. 다시 프로그램으로 돌아오면 폴더 안의 내용이 바뀌었다고 바뀐 리스트 (우리가 집어넣은 파일들)의 리스트가 뜹니다. 이것을 확인한 후에 summary 칸 안에 아무 단어를 적어주신 후 Commit 버튼을 누릅니다. 







3-10. 리스트가 사라진 것을 확인하셨다면 활성화된 Publish branch 버튼을 눌러주세요. 








3-11. 깃허브 홈페이지의 저장소에 방금 넣은 파일들이 뜬다면 GitHub에서의 작업은 완료되었습니다. 




깃허브 (GitHub) 정리

저장소(repository) 폴더 내의 내용 수정 후 

1. 바뀐 파일 목록 (Changes) 업데이트를 확인
2. Commit을 눌러 목록 업데이트 (수정된 파일이 있어요! 하고 GitHub에 말하는 거라 생각하시면 됩니다.) 
3. Publish branch 버튼을 눌러 진짜 GitHub 파일 업데이트. 

총 3번의 과정을 거치게 됩니다.

6. (선택) 게임 옵션 수정

이미지가 아닌 정말로 게임의 옵션 (리버스 수박게임 등...) 을 수정하고 싶으시다면... 큰 맘을 먹고 들어가봅시다. 

수정하셔야 할 파일은 src 폴더 내의 extraSetting.js 파일입니다. Setting 옵션은 아래와 같습니다. 

직접 수치를 수정하셔야 할 부분은 강조해두었습니다. 



사용한 에디터는 sublime Text3 입니다. 가볍게 코드 수정하기에 좋습니다. 사실 메모장에서도 불러올 수 있는데... 메모장에서 보기엔 눈이 아프니까... 



extraSetting 코드에서는 과일들을 번호로 관리합니다.  과일의 번호는 1. 리소스 만들기의 이미지를 참고해주세요. 




let firstFruit = 9; ▶ 가장 먼저 등장하는 떨어지는 과일을 지정할 수 있습니다. 바꾸고 싶을 경우 9 대신 0~10 의 바꾸고 싶은 과일 번호를 적습니다. 

let reverseLevelUp = false ; ▶ 리버스 (큰 과일->작은 과일) 모드와 오리지널 (작은 과일->큰 과일) 모드일지 조절할 수 있습니다. 리버스 모드를 원할 경우 false 대신 true를 적습니다. 


const minRandomFruitNum = reverseLevelUp ? 6 : 0; 

▲ 리버스 모드 때 떨어질 가장 작은 과일 번호 : 오리지널 모드때 떨어질 가장 작은 과일 번호

const maxRandomFruitNum = reverseLevelUp ? 11 : 5; 

▲ 리버스 모드 때 떨어질 가장 큰 과일 범위 : 오리지널 모드때 떨어질 큰 작은 과일 범위

이 숫자보다 1이 작은 숫자 과일이 떨어진다.
